# 为什么编译很慢？

## 原因

你的项目使用了 **Fyne GUI框架**，这是一个需要CGO的GUI库。编译时会：

1. **下载C依赖**：需要下载GLFW、OpenGL等C库
2. **编译C代码**：需要编译C/C++代码
3. **链接GUI库**：链接Windows GUI库

这个过程通常需要 **3-10分钟**，这是正常的。

## 已优化的措施

### 1. 添加了超时设置
```yaml
timeout-minutes: 10  # 每个服务最多10分钟
```

### 2. 添加了详细输出
```yaml
go build -v  # -v 参数显示编译进度
```

### 3. 添加了进度提示
```yaml
echo Building center_service (this may take 3-5 minutes for GUI apps)...
```

## 编译时间参考

- **第一个服务**：5-8分钟（需要下载C依赖）
- **后续服务**：3-5分钟（依赖已缓存）
- **总共**：约15-25分钟

## 如何加快编译

### 方法1：使用缓存（已自动）

GitHub Actions会自动缓存Go模块，第二次编译会快一些。

### 方法2：并行编译（可选）

可以修改工作流，让多个服务并行编译（但会增加复杂度）。

### 方法3：预编译依赖（复杂）

可以创建一个单独的job预编译C依赖，但这会增加工作流复杂度。

## 当前状态

编译慢是**正常的**，因为：
- ✅ GUI应用需要CGO
- ✅ 需要编译C/C++代码
- ✅ Windows环境需要下载GUI库

**建议**：耐心等待，通常5-10分钟就能完成。

## 如何查看进度

在GitHub Actions页面：
1. 点击运行记录
2. 展开 "Build center_service" 步骤
3. 可以看到实时编译输出
4. 看到 "Building..." 说明正在编译，不是卡住
